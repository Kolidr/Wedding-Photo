# Референсы свадебных фотографий

Светлый театральный сайт-портфолио с бордовыми и золотыми акцентами.
**5 сентября 2026 · Иркутск**

- 👀 **Просмотр — для всех, без пароля** (можно показывать фотографу с любого устройства)
- 🔒 **Загрузка, удаление и изменение размера — только по паролю**
- 🖼 Фото в общем виде обрезаются под своё окно; при открытии — **полный размер без обрезки**
- 🎚️ У **каждой фотографии** свои кнопки **«− / +»** — крупнее или мельче, чтобы собрать раскладку вручную (как коллаж)

---

## 🚀 Публикация (один раз)

1. Создайте репозиторий `Wedding-Photo` на GitHub — тип **Public**.
2. Загрузите в него файлы из архива:
   - `index.html`
   - `photos.json` (пустой список — нужен для старта)
   - папку `photos/`
   - `README.md` (необязательно)
3. **Settings → Pages → Source:** `Deploy from a branch`, **Branch:** `main` / `(root)` → **Save**.
4. Через 1–2 минуты сайт открывается по адресу:
   ```
   https://ВАШ-ЛОГИН.github.io/Wedding-Photo/
   ```
   Имя пользователя и репозиторий сайт **определит сам** из адреса.

---

## 🔑 Настройка доступа (один раз)

1. Создайте токен: https://github.com/settings/tokens/new — отметьте право **`repo`**, нажмите *Generate token*, скопируйте (`ghp_…`).
2. На сайте нажмите **⚙ (Настройки)**.
3. Вставьте **токен** и придумайте **пароль для входа**.
4. **Сохранить доступ.**

Токен будет зашифрован паролем (AES-GCM) и сохранён в репозитории как `auth.json`.
В открытом виде токен **нигде не лежит** — даже в коде сайта.

## 👰 Доступ для невесты

Невеста открывает тот же адрес → **Войти** → вводит **пароль** → может загружать, удалять и менять размер.
Токен ей не нужен — она работает с любого устройства, зная только пароль.

> Сменить пароль = заново открыть **⚙ Настройки**, ввести токен и новый пароль.

---

## 🖼 Как пользоваться

- **Войти** → **Добавить** (или перетащите файлы на галерею) — загрузка фото.
- Наведите на фото → **− / +** слева сверху — крупнее / мельче именно это фото.
- Наведите на фото → **корзина** справа снизу — удалить.
- Клик по фото → просмотр в полном размере, стрелки ← → листают.
- Раскладка (размеры фото) сохраняется в репозиторий — её видят все гости.

## 🔄 Как заменить файлы на GitHub (например, обновить `index.html`)

Когда приходит новая версия файла, старый нужно **перезаписать тем же именем**.

### Способ 1 — загрузить заново (проще всего)
1. Откройте свой репозиторий на GitHub.
2. Нажмите **Add file → Upload files**.
3. Перетащите новый файл (имя должно совпадать, например `index.html`).
   GitHub заменит старый файл этим же путём.
4. Внизу нажмите **Commit changes**.
5. Подождите 1–2 минуты — Pages обновит сайт.

> Папку `photos/` и файл `photos.json` заменять **не нужно** — там лежат ваши фотографии и раскладка. Обновляйте только `index.html` (и при желании `README.md`).

### Способ 2 — отредактировать прямо на сайте (для мелких правок)
1. В репозитории откройте нужный файл (клик по имени).
2. Нажмите **карандаш (✏️ Edit this file)** справа сверху.
3. Внесите изменения (или выделите всё и вставьте новое содержимое).
4. **Commit changes**.

### Если файл лежит в папке
Зайдите сначала в папку (например `photos/`), затем используйте те же кнопки
**Add file → Upload files** или **карандаш** — путь сохранится автоматически.

## ⏳ Про задержку

GitHub Pages кэширует файлы. После загрузки/удаления/изменения размера у **гостей**
изменения появляются обычно в течение 1–10 минут. У вас в текущем сеансе — сразу.

---

## ⚠️ Безопасность

- Зашифрованный `auth.json` лежит в публичном репозитории, но без пароля бесполезен —
  выбирайте **надёжный пароль**.
- Токен даёт доступ к вашим репозиториям. Если пароль или токен утёк —
  отзовите токен в настройках GitHub и настройте доступ заново.
- Пароль восстановить нельзя (он нигде не хранится) — только пересоздать доступ новым токеном.
