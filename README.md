# Референсы свадебных фотографий

Светлый театральный сайт-портфолио с бордовыми и золотыми акцентами.
**5 сентября 2026 · Иркутск**

- 👀 **Просмотр — для всех, без пароля**
- 🔒 **Загрузка, удаление, размер и порядок — по паролю**
- 🧩 Раскладка **автоматическая** (как иконки на телефоне): фото сами встают без пустот
- 🖼 Форматы фиксированные: 1×1, 2×1, 1×2, 2×2, 3×1, 1×3, 3×2, 2×3, 3×3 + **Авто**
- ✂️ Для каждого фото: **обрезать** под формат или показать **целиком с фоном**
- ↔️ Перестановка фото стрелками, видно позицию (например «4 из 12»)
- 📱 Можно **добавить на экран iPhone** как приложение
- 🔑 Пароль **запоминается** на устройстве (с возможностью выйти)

## 🚀 Публикация (один раз)

1. Создайте репозиторий `Wedding-Photo` на GitHub — тип **Public**.
2. Загрузите в него **все** файлы из архива:
   - `index.html`
   - `photos.json`
   - `manifest.webmanifest`
   - `icon-180.png`, `icon-192.png`, `icon-512.png`
   - папку `photos/`
   - `README.md` (необязательно)
3. **Settings → Pages → Source:** `Deploy from a branch`, **Branch:** `main` / `(root)` → **Save**.
4. Через 1–2 минуты сайт открывается по адресу:
   ```
   https://ВАШ-ЛОГИН.github.io/Wedding-Photo/
   ```
   Имя пользователя и репозиторий сайт определит сам из адреса.

---

## 🔑 Настройка доступа (один раз)

1. Создайте токен: https://github.com/settings/tokens/new — отметьте право **`repo`**, *Generate token*, скопируйте (`ghp_…`).
2. На сайте нажмите **⚙ (Настройки)** → вставьте **токен** и придумайте **пароль** → **Сохранить доступ**.

Токен шифруется паролем (AES-GCM) и хранится в репозитории как `auth.json`. В открытом виде нигде не лежит.

---

## 👰 Доступ для невесты

Открыть тот же адрес → **Войти** → ввести **пароль** (галочка «Запомнить на этом устройстве» оставляет вход) → можно загружать, удалять, менять размер и порядок. Токен не нужен.

> Выйти можно кнопкой **Выйти** — пароль с устройства удалится.
> Сменить пароль = заново открыть **⚙ Настройки**, ввести токен и новый пароль.

---

## 🖼 Как пользоваться

- **Войти** → **Добавить** (или перетащите файлы) — загрузка. Новые фото встают **в конец**.
- Раскладка собирается сама, без пустых мест. Наведите на фото (или коснитесь на телефоне) — появятся кнопки:
  - **‹ ›** по бокам — переместить фото левее (ближе к началу) или правее.
  - **⤢** — выбрать формат: Авто или фиксированный (1×1, 3×1, 1×3, 3×3, 2×3, 3×2 и т.д.).
  - **▢ / ▣** — переключить: обрезать под формат ↔ показать фото целиком (с фоном по краям).
  - **корзина** — удалить.
  - **Открыть** — фото на весь экран без обрезки, стрелки ← → листают.
- **Авто** = фото показывается целиком, без обрезки, размер подбирается сам.
- Номер у фото (например «4 из 12») показывает позицию и общее количество.
- Все изменения сохраняются в репозиторий — их видят все гости.

## 📱 Добавить на экран iPhone (как приложение)

1. Откройте сайт в **Safari**.
2. Нажмите кнопку **«Поделиться»** (квадрат со стрелкой вверх).
3. Выберите **«На экран Домой»** → **Добавить**.

Появится иконка с золотым `&` на бордовом фоне; сайт откроется на весь экран, как приложение.
(На Android — меню браузера → «Установить приложение» / «Добавить на главный экран».)

## 🔄 Как заменить файлы на GitHub (например, обновить `index.html`)

### Способ 1 — загрузить заново (проще всего)
1. Откройте репозиторий → **Add file → Upload files**.
2. Перетащите новый файл с тем же именем (например `index.html`) — он заменит старый.
3. Внизу **Commit changes**. Через 1–2 минуты Pages обновит сайт.

> Папку `photos/`, `photos.json` и `auth.json` заменять **не нужно** — там фото, раскладка и доступ.

### Способ 2 — отредактировать на сайте (для мелких правок)
Откройте файл → **карандаш (✏️ Edit)** → измените → **Commit changes**.

## ⏳ Про задержку

GitHub Pages кэширует файлы. После изменений у **гостей** они появляются в течение 1–10 минут. У вас в текущем сеансе — сразу.

---

## ⚠️ Безопасность

- Зашифрованный `auth.json` лежит в публичном репозитории, но без пароля бесполезен — выбирайте **надёжный пароль**.
- Галочка «Запомнить» хранит доступ на устройстве. На чужом устройстве не отмечайте её или нажмите **Выйти** после работы.
- Если пароль/токен утёк — отзовите токен в настройках GitHub и настройте доступ заново.
