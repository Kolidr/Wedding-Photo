# Референсы свадебных фотографий

Сайт-альбом с примерами кадров, которые нравятся паре, — чтобы показать фотографу и обсудить будущую съёмку. Спутник сайта-приглашения.

**Просмотр, скачивание и комментарии — для всех. Загрузка, удаление, изменение размера и порядка — только по паролю.**

---

## Что умеет

- **Авто-мозаика.** Плитки складываются как на главном экране телефона, без пустот; ориентация определяется автоматически.
- **Любой размер плитки.** Для каждого фото — «Авто» или фиксированный формат: 1×1, 2×1, 1×2, 2×2, 3×1, 1×3, 3×2, 2×3, 3×3.
- **Два режима заполнения.** «Заполнить (обрезать)» — по умолчанию, фото заполняет плитку; «Полностью (с фоном)» — фото видно целиком.
- **Открытие фото на весь экран** с панелью управления: листание, изменение порядка, формат, скачивание, удаление.
- **Комментарии к каждому фото** — может оставить любой, даже без пароля. Имя спрашивается один раз и запоминается. Если у фото есть комментарий, на плитке внизу появляется значок; сами комментарии видно при открытии фото.
- **Скачивание.** Любое фото — кнопкой «Скачать» при открытии. Весь альбом — кнопкой «Скачать все» (архивом, работает и на телефоне).
- **Изменение порядка** стрелками. При перемещении фото остаётся активным — можно двигать дальше; в открытом фото есть кнопки «в начало» и «в конец».
- **Установка как приложение** на экран «Домой» (iPhone/Android) со своей иконкой.
- Данные хранятся в этом же репозитории, отдельный сервер не нужен.

## Управление (для владельца)

### Поведение нажатий
- **Гость** (без пароля): нажатие на фото — сразу открывает его.
- **Компьютер** (с паролем): наведение — увеличение и инструменты; клик — открывает фото.
- **Телефон** (с паролем): первое нажатие — выбирает фото и показывает инструменты; второе — открывает фото.

### Первая настройка доступа — один раз
1. Откройте сайт → шестерёнка (⚙) вверху справа.
2. Создайте **Personal Access Token** на GitHub с правом **repo**: <https://github.com/settings/tokens/new>
   (Срок — на ваш выбор; **Classic**-токен, галочка `repo`.)
3. Вставьте токен и **придумайте пароль**. Нажмите «Сохранить доступ».

Токен **шифруется вашим паролем** (AES-GCM) и хранится в репозитории в файле `auth.json`. В открытом виде он нигде не записывается. Дальше на любом устройстве — просто «Войти» с паролем.

### Добавить / удалить фото
- **Войти** → **Добавить** (или перетащите файлы в окно на компьютере).
- Удалить: в открытом фото — кнопка с корзиной (нужен пароль).

## Установка как приложение (iPhone)

1. Откройте сайт в **Safari**.
2. Кнопка «Поделиться» → **«На экран „Домой"»**.
3. Появится иконка. Сайт откроется на весь экран, как приложение.

(На Android — меню браузера → «Добавить на главный экран».)

## Запомнить вход / выйти

- При входе галочка **«Запомнить на этом устройстве»** — больше пароль вводить не нужно (на этом устройстве).
- Кнопка **«Выйти»** убирает доступ и возвращает режим просмотра.

## Как заменить файл сайта на GitHub

Чтобы обновить сайт (например, новую версию `index.html` или иконки):

**Способ 1 — загрузить с тем же именем (просто):**
1. Откройте репозиторий на github.com.
2. **Add file → Upload files**.
3. Перетащите файлы (`index.html`, `icon-*.png`, `manifest.webmanifest`) — имена должны совпадать.
4. Внизу **Commit changes**. Старые файлы заменятся.

**Способ 2 — отредактировать текстом:**
1. Откройте нужный файл → значок карандаша (**Edit**).
2. Вставьте новое содержимое → **Commit changes**.

> После обновления у гостей сайт может обновиться не сразу (кэш GitHub Pages, 1–10 минут). У вас как у владельца изменения видны сразу.

## Частые вопросы

**Гость оставил комментарий — увижу ли я его?**
Комментарии, оставленные **с паролем**, сохраняются в альбом и видны всем. Комментарии гостя **без пароля** сохраняются только на его устройстве (помечены «только у вас») — так устроено, потому что сайт без отдельного сервера и запись в альбом возможна только с доступом.

**Фото открывается, но не скачивается / открывается в новой вкладке.**
Кнопка «Скачать» сохраняет файл напрямую. Если что-то не так — проверьте, что сайт открыт по адресу `…github.io/…`, а не как локальный файл.

**Безопасно ли хранить токен?**
Токен зашифрован паролем и без него бесполезен. Любой может *смотреть* альбом, но *менять* — только знающий пароль. При утере доступа создайте новый токен и заново пройдите настройку.
